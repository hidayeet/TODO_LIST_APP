# TO DO WEB APP
this project isbuilt using REACT and MATERIAL UI
the app follows the CRUD principle (create,read,update,delete)
it aslo includes a DARK MODE and LIGHT MODE

## FEATURES
- add new task 
- edit existing tasks
- delete tasks 
- mark tasks as completed
- dark mode and light mode 

## TECK STACK
the project was built using the following technlogies:
- react (v18)
- javascript (E26)
- material UI (MUI 5)
- material UI icons
- styling with material ui 'sx' system
- node.js (runtime environment)
- npm(pachage manager)
- create react app

## INSTALLATION & SETUP
Follow these steps to run the project locally on your computer
### Requirements
Make sure you have the following installed on your system:
- Node.js
- npm
### 1.dowload the project
dowload the project from GITHUB and extract the zip file
### 2. Open a terminal
Open a "Terminal" OR "Command Prompt" on your computer
### 3. Navigate to the project folder
use the "cd" (change directory) is used to enter a folder from the terminal
exemple:
cd todo-list-app
### 4. Install the required dependencies
with the following command:
npm install
This command installs all the libraries used in the project
### 5. Run the application
with the command :
npm start
### 6. Open the application in your browser
The application will start automatically.  
If it does not open, go to:
http://localhost:3000

## PROJECT STRUCTURE
The project is organized as follows:
- public  
Contains static files used by the application such as the main HTML file
- src  
Contains the main source code of the React application

Inside the src folder:
- App.js  
The main component that loads the application
- todoapp.js  
Contains the main logic of the To Do List such as adding, editing, and deleting tasks
- index.js  
The entry point of the React application that renders the main component
- index.css  
Contains global styles for the application
Other important files:
- package.json  
Contains project dependencies and scripts used to run the application
- package-lock.json  
Ensures consistent versions of installed dependencies
- .gitignore  
Specifies files and folders that Git should ignore
- README.md  
Documentation that explains the project and how to run it
